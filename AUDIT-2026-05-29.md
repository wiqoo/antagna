## ⚡ Performance plan (ranked)

1. **Collapse per-row `has_permission()` in `v_*_safe` views** (the biggest masking-cost win,
   and tied to the reverted hang). Resolve the caller's permission booleans **once per request**
   (e.g. set `app.perm_*` GUCs in `withProfileScope`, views read `current_setting(...)::bool`)
   instead of a function-body lookup per masked column per row. **Required before re-wiring the
   page switchover.** Also verify indexes on `position_default_permissions(position_key,permission_key)`.
2. **CRM landing: ~200 correlated subquery executions/load** (`crm/page.tsx:72-95`). Replace the
   3 per-row correlated subqueries + correlated `ORDER BY MAX()` with one `LEFT JOIN LATERAL`/CTE.
3. Re-run `get_advisors(performance)` after the authz/transaction fixes to catch hot-path indexes.

## 🎨 UX/UI improvements (ranked)

1. **Silent action failures** — `{ok:false}` returns discarded by inline `<form action>` wrappers;

   `useActionState` + render errors, or throw on hard failure. (highest UX impact)
2. `addContact` returns silently on empty name (form resets, no message). Structured error.
3. Project-detail equipment `<select>` has a dead disabled optgroup ("—— أو مجموعة ——") — group
   reservations impossible despite `addReservation(groupId)` support. Wire it or delete it.
4. `/clients/new` code-slug uses `[A-Za-z]` only → Arabic-only names yield literal `CLNT`; dedupe
   caps at 99 then silently collides. Transliterate or sequential `CLNT-NNN`.
5. Wizard PM seat ≠ "team member" — setting a PM in the wizard shows "فريق المشروع 0"; clarify or
   auto-create the assignment row.
6. Restricted-access UX: masked pages show "—" everywhere with no explanation — add a "you don't
   have access to financials" affordance instead of silent blanks (when masking is re-wired).
